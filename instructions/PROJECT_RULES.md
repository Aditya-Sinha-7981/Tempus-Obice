# PROJECT_RULES.md

## General

- Read AI_CONTEXT.md before making any changes.
- Read NEXT_TASK.md before beginning work.
- Complete only the current task.
- Do not skip ahead.
- Do not implement future phases.

---

## Code Quality

- Produce production-ready code.
- No placeholder implementations unless explicitly requested.
- No TODO comments unless the task specifically requires them.
- Keep code readable.
- Prefer maintainability over cleverness.

---

## Architecture

- Do not introduce new dependencies.
- Do not change the folder structure.
- Do not replace technologies.
- Do not add Tailwind.
- Do not add state management libraries.
- Do not add routing libraries.
- Do not add backend code.

---

## Workflow

For every completed task:

1. Verify the project builds.
2. Verify TypeScript passes.
3. Explain what changed.
4. Update NEXT_TASK.md if instructed.
5. Never continue automatically to another phase.

---

## When Uncertain

If requirements are ambiguous:

Stop.

Explain the ambiguity.

Wait for approval.

Never guess architecture.
